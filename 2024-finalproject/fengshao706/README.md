# **课程学习总结**

  通过本课程的学习，我初步了解了构建和维护现代软件项目所必需的基础工具链，包括 **Shell脚本**、**Git版本控制**、**CMake构建系统** 以及 **Linux的一些基本系统架构**。而这些通常是课本所不会教的，但它却非常重要，这正如本次课程的主题：the missing semester----计算机教育缺失的一课

## **一、 知识体系梳理与核心收获**

### **1. Linux系统基本架构**

**理解**：学习了Linux系统的核心组成部分，如内核、Shell、进程管理和用户权限等。

**收获**：这为我理解其他工具的运行环境打下了坚实基础。我明白了linux的软件安装在哪里，内核kernel和shell的关系是什么，linux怎么创建进程和结束进程。对进程和权限的理解，使得我能够安全、高效地在linux系统上开发，也将为接下来的ros学习打下坚实的基础。

### **2. Shell脚本**

**理解**：Shell不仅是输入命令的界面，更是一个强大的脚本编程环境和自动化工具。我掌握了变量、条件判断、循环、函数以及管道、重定向等核心概念。

**收获**：我能够编写脚本来自动化重复性任务，例如：获取系统近10次的平均开机时间，能够使用`grep`,`sed`等工具快速提取分析修改文本信息，能够对文件进行批量化处理。

### **3. Git**

**收获**：掌握了Git的分布式版本控制核心概念，包括工作区、暂存区、仓库、提交、分支、合并等，能够较为熟练地使用git的一些常用命令来做代码备份和团队合作，例如一些日常的工作流，像`git add`,`git commit`,`git push`,`git status`,`git log`等命令，并能区分工作分支和主分支，能够熟练地创建，切换，合并分支，并解决可能存在的分支冲突，最后，我能够使用github等远程仓库上进行有效的团队合作，这将极大地提升自己的开发效率和开发体验

### **4. CMake**

**理解**：认识到在C/C++等语言中，直接使用命令行编译大型项目非常复杂。CMake是一个**元构建系统**，它通过一个平台无关的 `CMakeLists.txt` 文件来生成适合当前平台（如Unix的Makefile或Windows的Visual Studio项目）的构建文件。

**收获**：学会了编写基本的 `CMakeLists.txt`，包括定义项目名、指定C++标准、添加可执行文件和目标链接库。

## **二、 工具间的协同效应**

本课程最大的价值在于让我看到了这些工具是如何协同工作的，构成了一个完整的开发闭环：

1. **环境与基础**：我工作在 **Linux** 系统上，这是一个稳定、高效的开发环境。
2. **编码与版本管理**：我用编辑器写代码，并通过 **Git** 来管理代码的所有版本变更。
3. **构建与测试**：当我完成一部分功能后，我使用 **CMake** 来配置和构建我的项目，生成可执行文件或库。
4. **自动化与部署**：我可以编写 **Shell脚本**，将构建、测试、打包、部署等一系列步骤自动化。这个脚本本身也可以被 **Git** 管理起来。
